# Phase 1: Web 管理界面 — 产品设计文档

> 版本: v1.0 | 日期: 2026-03-27 | 状态: 设计中

## 1. 产品概述

### 1.1 目标用户

| 用户角色 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| **DBA（数据库管理员）** | 日常备份管理、故障恢复、数据验证 | 可靠性、操作效率、详细的日志和状态 |
| **运维工程师** | 监控备份状态、处理告警、执行恢复 | 一目了然的状态总览、快捷操作 |
| **技术负责人** | 查看备份策略、审批操作、审计 | 全局视角、权限管控、操作可追溯 |

### 1.2 核心价值

- **降低备份管理门槛**：可视化操作替代命令行/API 调用，非专业人员也能完成基本操作
- **提升运维效率**：仪表盘实时总览，批量操作，Cron 可视化编辑，减少重复劳动
- **增强风险感知**：备份成功率趋势、存储空间预警、恢复验证状态一目了然
- **操作可追溯**：完整的操作日志和审计记录

### 1.3 设计原则

1. **简洁优先**：信息密度适中，默认视图只展示关键指标，详情按需展开
2. **专业可靠**：深色/浅色主题切换，状态色彩语义明确（绿=成功、红=失败、黄=警告）
3. **响应式设计**：适配桌面（主要）和平板（辅助），最小支持 1024px 宽度
4. **渐进增强**：核心功能不依赖 WebSocket，实时日志为增强体验

```

│              │  └────────────────┘  │                │
│              │                      │                │
│              │  [  登  录  ]        │                │
│              │                      │                │

│              └──────────────────────┘                │
│                                                     │
```

**交互流程**：
1. 用户输入 API Key → 点击登录

**交互流程**：
1. 填写表单 → 可随时点击"测试连接"验证数据库连通性

3. 保存 → 调用 `POST /api/v1/jobs`（新建）或 `PUT /api/v1/jobs/:id`（编辑）
4. 保存成功 → Toast 提示 → 返回列表

**Cron 可视化编辑器**：

- 预设快捷选项：每小时、每天、每周、每月
- 5 字段独立选择（分/时/日/月/周）
- 实时预览"下次执行时间"（调用后端计算）
- 手动输入模式（高级用户）

**API 对接**：
- `POST /api/v1/jobs` — 创建
- `PUT /api/v1/jobs/:id` — 更新
- `GET /api/v1/jobs/:id` — 获取详情（编辑回填）

**⚠️ API 增量需求**：
- `GET /api/v1/jobs` 需增加查询参数：`search`（任务名模糊搜索）、`database_type`、`enabled`
- 新增 `POST /api/v1/jobs/:id/test-connection` — 测试数据库连接
- 新增 `GET /api/v1/jobs/:id/next-runs?cron=...&limit=5` — 计算下次执行时间

## 3. 技术选型建议

### 3.1 前端框架

**推荐：React 18 + TypeScript + Vite**

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| React + TypeScript | 生态丰富、类型安全、组件化 | 学习曲线略高 | ⭐⭐⭐⭐⭐ |
| Vue 3 + TypeScript | 上手快、中文社区好 | 大型项目工程化稍弱 | ⭐⭐⭐⭐ |
| Next.js (React SSR) | SEO、首屏快 | 过度设计（内部系统不需要） | ⭐⭐⭐ |

### 3.2 UI 组件库

**推荐：Ant Design 5.x**

- 企业级组件丰富（Table、Form、Drawer、Steps 等直接可用）
- 中文文档完善
- 内置暗色主题
- TypeScript 支持好

备选：Arco Design（字节出品，更现代）

### 3.3 状态管理

**推荐：Zustand + React Query (TanStack Query)**

| 层 | 方案 | 职责 |
|----|------|------|
| 服务端状态 | TanStack Query | API 请求缓存、自动刷新、乐观更新 |
| 客户端状态 | Zustand | 用户偏好、主题、侧边栏状态 |
| 表单状态 | React Hook Form | 表单校验、提交管理 |

### 3.4 WebSocket 集成

**方案：自定义 Hook 封装 WebSocket 连接**

```typescript

function useBackupLogs(jobId: string) {
  // 自动连接 ws://host/ws/logs/:job_id
  // 自动重连（指数退避）
  // 返回 { logs, isConnected, error }

```

- 用于实时日志推送（任务执行时）
- 仪表盘状态实时更新
- 连接断开时自动降级为轮询

## 5. 页面路由设计

```
/                       → 重定向到 /dashboard
/login                  → 登录页（API Key 模式）

/dashboard              → 仪表盘首页

/jobs                   → 任务列表
/jobs/new               → 新建任务
/jobs/:id/edit          → 编辑任务

/records                → 备份记录列表
/records/:id            → 记录详情（侧边抽屉，URL 变化但不独立页面）

/restore                → 恢复向导
/restore/history        → 恢复历史

/verify                 → 备份验证

/settings               → 系统设置
/settings/storage       → 存储配置
/settings/notify        → 通知配置
/settings/general       → 通用设置
/settings/about         → 关于

/*                      → 404 页面
```

**路由守卫**：
- 未认证用户 → 重定向到 `/login`
- 已认证用户访问 `/login` → 重定向到 `/dashboard`
- Token 过期 → 自动刷新 → 失败则跳转 `/login`
