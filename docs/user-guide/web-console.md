# Web 控制台使用指南

本文档详细介绍 db-backup Web 控制台的使用方法。

## 访问控制台

### 启动服务

```bash
# 启动服务
./db-backup -config configs/config.yaml

# 服务默认运行在 8080 端口
```

### 访问地址

```
http://localhost:8080
```

首次访问时需要登录。

---

## 登录与认证

### 默认账户

首次安装时，系统会创建默认管理员账户：

| 字段 | 默认值 |
|------|--------|
| 用户名 | `admin` |
| 密码 | `admin123` |

⚠️ **首次登录后请立即修改密码！**

### 修改密码

1. 点击右上角头像
2. 选择「个人设置」
3. 修改密码并保存

## 📊 Dashboard

Dashboard 是登录后的默认页面，展示系统整体状态。

### 主要内容

| 区域 | 说明 |
|------|------|
| **统计卡片** | 任务总数、正在运行、今日备份、存储使用 |
| **任务状态** | 运行中/成功/失败任务数量 |
| **最近告警** | 最新告警列表 |
| **最近备份** | 最近 10 条备份记录 |

### 操作

- 点击任务状态卡片可跳转到相应列表
- 点击备份记录可查看详情
- 点击告警可查看处理建议

## 📋 任务管理

### 任务列表

展示所有备份任务，支持以下操作：

| 操作 | 说明 |
|------|------|
| **立即执行** | 手动触发一次备份 |
| **编辑** | 修改任务配置 |
| **删除** | 删除任务（需确认） |
| **启用/禁用** | 控制任务是否参与调度 |

### 创建任务

点击「新建任务」按钮，进入任务创建向导：

#### 第一步：基本信息

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 任务名称 | 任务的显示名称 | ✅ |
| 数据库类型 | MySQL/PostgreSQL/MongoDB/SQL Server/Oracle | ✅ |
| 描述 | 任务描述（可选） | |

#### 第二步：数据库配置

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 主机地址 | 数据库服务器地址 | ✅ |
| 端口 | 数据库端口 | ✅ |
| 用户名 | 连接数据库的用户名 | ✅ |
| 密码 | 连接密码 | ✅ |
| 数据库名 | 要备份的数据库名 | |

#### 第三步：备份配置

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 备份模式 | 全量/增量 | ✅ |
| 调度表达式 | Cron 表达式 | ✅ |
| 压缩 | 是否启用压缩 | |

#### 第四步：存储配置

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 存储类型 | 本地/S3/OSS/COS | ✅ |
| 存储路径 | 备份文件存放位置 | ✅ |

#### 第五步：加密配置（可选）

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 启用加密 | 是否加密备份文件 | |
| 加密密钥 | 32 字节 hex 字符串 | |
| 密钥环境变量 | 从环境变量读取密钥 | |

#### 第六步：通知配置（可选）

| 字段 | 说明 |
|------|------|
| 钉钉 Webhook | 钉钉群机器人地址 |
| 飞书 Webhook | 飞书群机器人地址 |
| 邮件通知 | 接收通知的邮箱 |
| 告警规则 | 失败/超时等条件触发通知 |

### 编辑任务

点击任务列表中的「编辑」按钮，可修改任务配置。

### 立即执行

点击「立即执行」按钮，系统会立即触发一次备份任务，同时打开实时日志窗口。

## 🔄 数据恢复

### 恢复向导

点击「数据恢复」进入恢复流程：

#### 第一步：选择备份

从备份记录中选择要恢复的备份，或直接输入备份文件路径。

#### 第二步：目标配置

| 字段 | 说明 | 必填 |
|------|------|:----:|
| 目标主机 | 恢复到的数据库服务器 | ✅ |
| 目标端口 | 恢复到的数据库端口 | ✅ |
| 目标数据库 | 恢复到的数据库名 | ✅ |
| 目标用户 | 连接用户名 | ✅ |
| 目标密码 | 连接密码 | ✅ |

#### 第三步：恢复选项

| 选项 | 说明 |
|------|------|
| 覆盖已有数据 | 是否覆盖目标数据库（SQL Server） |
| 恢复模式 | recovery / norecovery / standby |

#### 第四步：确认执行

阅读恢复说明，确认无误后点击「开始恢复」。

### 异机恢复

如果目标服务器与备份源不同，选择「异机恢复」，系统会自动处理路径差异。

### 审批流程

恢复敏感数据库可能需要审批流程，详见 [审批管理](approval.md)。

## 📦 存储管理

### 存储列表

展示已配置的存储后端：

| 存储 | 类型 | 状态 | 容量 |
|------|------|------|------|
| 本地存储 | local | ✅ 正常 | 100GB/500GB |
| S3 备份 | s3 | ✅ 正常 | — |

### 存储详情

点击存储可查看：

- 存储使用统计
- 备份文件列表
- 容量预警设置

### 添加存储

支持添加多种类型的存储后端：

1. **本地存储**
   - 路径：本地磁盘目录
   - 权限：确保运行用户有读写权限

2. **S3/MinIO**
   - Endpoint、S3 Bucket、AK/SK

3. **阿里云 OSS**
   - Endpoint、OSS Bucket、AK/SK

4. **腾讯云 COS**
   - Region、Bucket、SecretID/SecretKey

## 📝 审批管理

对于敏感操作（如生产环境恢复），可启用审批流程。

### 审批流程

```
申请 → 审批人审批 → 执行/拒绝
```

### 申请恢复

1. 在「数据恢复」中选择「需要审批」
2. 填写恢复原因
3. 提交申请

### 审批操作

审批人收到通知后：
- **通过**：继续恢复操作
- **拒绝**：说明拒绝原因

## 🔧 故障排查

### 无法登录

1. 检查服务是否运行：`curl http://localhost:8080/health`
2. 检查账户密码是否正确
3. 检查是否有登录锁定

### 任务无法执行

1. 检查数据库连接是否正常
2. 检查备份工具是否安装（mysqldump 等）
3. 查看任务日志获取详细错误

### 存储连接失败

1. 检查网络连通性
2. 检查凭证是否正确
3. 检查存储权限

### 实时日志无法显示

1. 检查 WebSocket 是否被防火墙拦截
2. 检查 Origin 白名单配置
